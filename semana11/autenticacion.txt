El login permite identificar a un usuario antes de darle acceso a ciertas funciones dentro de la plataforma. Gracias a la autenticación es posible controlar quién puede ingresar y qué acciones puede realizar cada persona. Algunas rutas deben estar protegidas porque pueden contener información privada o registros internos. También pueden permitir modificaciones importantes dentro del sistema. No se deben exponer públicamente contraseñas, documentos de identidad, teléfonos, direcciones ni correos electrónicos. Tampoco deben compartirse tokens, claves de API ni archivos de configuración sensibles. La protección de esta información fortalece la seguridad y la confianza de la comunidad.
